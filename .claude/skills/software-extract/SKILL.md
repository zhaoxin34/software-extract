# software extract

用于抽取软件中的精华部分，生成提示词，提示词可用于创建新的工程。

# 工作流程

1. 用自然语言获得用户的分析目标目录，如果github url，则先要clone项目到临时文件夹

> notice: 临时文件夹用当前项目的temp目录
> notice：如果git项目，如果用户指定了分支，需要先切换到指定的分支, 如果包含未提交内容，导致不能切换分支的时候，需要提示用户提交或stash代码

2. 进入目标目录，大致了解项目的结构，大致了解项目的意图, 然后生成一个初级的报告, 这个分析报告主要用于接下来生成分析任务

> notice: 这步要判断项目的价值，如果遇到项目质量特别低，比如项目下就没有可以用于创建新工程的内容，则提示用户，然后退出

3. 创建抽取的结果存放目录

> notice: 抽取结果存放目录 = {当前目录}/raw/{待抽取目标项目名称}

4. 存放上面生成初级分析报告

> notice: 初级分析报告文件名 = primary-report.md

5. 根据初级分析报告生成分析结果目录和分析任务

有时，要分析的工程可能是一个超大工程，可能是由很多系统、子系统、模块、子模块构成的，我们先假设这个项目是一个最复杂的情况，那么可能生成的抽取报告的结构大致如下

```yaml
系统1:
  子系统1:
    模块1:
      子模块1:
        Feature1:
          Spec1:
          Spec2:
        Feature2:
          Spec1:
    模块2:
      Feature2:
        Spec1:
  子系统2:
    模块1:
      子模块1:
        Feature2:
          Spec1:
系统2:
  模块1:
    Feature2:
      Spec1:
  模块1:
    Feature2:
      Spec1:

```

> [!NOTE] 结构不是死的，他可以是简单的样子，比如 模块 -> 子模块，也可以是 系统 -> 模块

> [!NOTE] 最后一个级别必须是模块或子模块，因为要在模块下放置Feature list

> [!NOTE] 一个Feature可能会由多个spec构成
